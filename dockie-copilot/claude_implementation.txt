## 4. Atomic conditional upsert for position saves

**The problem:** `PositionRepository.save()` does `SELECT` then conditional `INSERT` — two round trips, and between them another worker could insert a newer position, leaving a race condition. Under a streaming AIS feed with one goroutine per vessel this serialises writes for each MMSI.

**What to do:**

Replace the read-then-write with a single PostgreSQL `INSERT ... ON CONFLICT DO UPDATE WHERE` that enforces monotonicity at the database level. PostgreSQL's `ON CONFLICT` with a `WHERE` clause on the update is atomic.

You need a unique constraint on `(mmsi)` for the latest position, but the positions table stores history too — so the right approach is either a partial unique index on the latest position, or (simpler) just using the conditional update and trusting the DB constraint to prevent stale overwrites.

The cleanest model: keep history in the positions table, but add a separate `latest_positions` table (or materialised view) that holds exactly one row per MMSI, updated atomically. For the MVP scope, a conditional upsert on a `latest_positions` table is simpler than a partial index on `positions`.

**Prompt for Codex:**

> Refactor `PositionRepository.save()` in `position_repo.py` and add the supporting migration as follows:
>
> 1. In `orm.py`, add a new ORM model `LatestPosition` mapping to a table `latest_positions`. It should have the same columns as `Position` except no `id` primary key — instead use `mmsi` as the primary key (String 20, not nullable). Include all position fields: `imo`, `vessel_name`, `latitude`, `longitude`, `geom`, `sog_knots`, `cog_degrees`, `heading_degrees`, `navigation_status`, `destination_text`, `source`, `observed_at`, `raw_event_id`, `created_at`, `updated_at` (server default now, onupdate now).
>
> 2. In `position_repo.py`, replace the `save()` method body with a PostgreSQL `INSERT INTO latest_positions (...) VALUES (...) ON CONFLICT (mmsi) DO UPDATE SET ... WHERE latest_positions.observed_at < EXCLUDED.observed_at`. Use `sqlalchemy.dialects.postgresql.insert` for this. This makes the staleness check atomic and eliminates the read round trip. Return the resulting row.
>
> 3. Add a new method `get_latest_for_mmsi(mmsi: str) -> LatestPosition | None` that queries `latest_positions` instead of `positions`. Update `get_latest_for_imo`, `get_latest_for_mmsis`, `get_latest_for_imos` to query `latest_positions` for current state lookups.

> 4. Keep `save_raw()` writing to `positions` unchanged — history still accumulates there.
>
> 5. Create an Alembic migration that: creates the `latest_positions` table, adds a GiST index on `geom`, and adds a btree index on `observed_at`. Add a comment: `-- Stores exactly one row per MMSI; updated atomically via conditional upsert`.
